# experiments/ — 보조 실험 (공개 간판 아님)

이 폴더는 **진단 도구의 본편이 아닙니다.** "이 노선을 신설하면 어떻게 될까" 같은 처방 시뮬레이션 실험이며,
참고용·역량 기록용으로 보존합니다. **본편은 진단(`../accessibility.py`)입니다.**

> 왜 본편에서 뺐나: "내가 만든 노선이 더 낫다"는 처방형 결과물은 중립 공공재의 메시지를 흐리고
> 정책 논쟁으로 빨려 들어갑니다. 이 프로젝트는 **"어디가 왜 불리한가"를 보여주는 진단**에 집중합니다.

## 파일
- `scenario_search.py` — 고수요·원거리 앵커 → 거점 급행 코리도를 N개 자동생성·평가(수요가중 접근 단축 랭킹). config 기반 병렬 엔진.
- `scenario_results.csv` — 위 100개 시나리오 결과(⚠️ 목적지 "시청 단일" 기준 → 다목적지 미반영. 본편 진단 기준과 다름).
- `whatif_express.py` — 단일 급행 노선 What-if 시뮬(동구 급행 예시).

## ⚠️ 한계
처방 시뮬은 본편보다 가정이 더 많습니다(급행 평속 30km/h, 차량·기사·기존노선 중복 미고려). [../LIMITATIONS.md](../LIMITATIONS.md) 참조.
